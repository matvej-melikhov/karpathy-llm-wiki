# help: справка по скиллам wiki

Read-only скилл. Никаких записей, никаких побочных эффектов. Цель — показать
пользователю, что доступно в этом vault, и при необходимости детально описать
один конкретный скилл.

---

## Команда

| Команда | Поведение |
|---|---|
| `/help` | Таблица всех скиллов из `.claude/skills/`: имя, одно-строчное описание, основные триггеры. |
| `/help <команда>` | Подробная справка по одному скиллу: description + «Команда и флаги» + триггеры. |
| `/help all` | То же что `/help` (alias). |

`<команда>` — имя без слэша (`/help ingest`, не `/help /ingest`). Слэш в начале — терпим, обрезай.

---

## Workflow

### Без аргумента — общий обзор

1. `Glob` по `.claude/skills/*/SKILL.md` — список всех скиллов в проекте.
2. Для каждого SKILL.md прочитай frontmatter: достань `name`, `description`.
3. Из description вытащи короткое summary (первое предложение) и триггеры
   (всё после слова «Триггеры:» или «Triggers:» — обычно последний абзац).
4. Выведи таблицу:

   ```markdown
   | Команда | Что делает | Основные триггеры |
   |---|---|---|
   | `/<name>` | <первое предложение description> | `/name`, `триггер1`, `триггер2` |
   ```

5. В конце короткая подсказка: «Подробнее: `/help <команда>`».

Порядок строк — алфавитный по имени скилла. `help` тоже включай — он такой же скилл.

### С аргументом — детальная справка

1. Нормализуй аргумент: убери ведущий `/`, приведи к нижнему регистру.
2. `Read` файл `.claude/skills/<arg>/SKILL.md`. Если не существует —
   `Glob` по `.claude/skills/*/SKILL.md`, найди похожие имена (Levenshtein или
   substring match) и предложи 2-3 ближайших варианта.
3. Из найденного SKILL.md извлеки и покажи:
   - `name` и `description` целиком (из frontmatter).
   - Раздел «Команда» / «Команда и флаги» / «Команды» — первую таблицу с командами и описанием.
   - Триггеры — обычно перечислены в конце description; если есть отдельный раздел или подзаголовок «Триггеры», возьми из него.
4. В конце — ссылка-указание, где лежит сам скилл (`.claude/skills/<arg>/SKILL.md`),
   чтобы пользователь мог открыть полный текст при желании.

Формат вывода:

```
## /<name>

<description>

### Команды
<таблица команд и флагов из SKILL.md>

### Триггеры
<список триггеров одной строкой>

Полный SKILL.md: `.claude/skills/<name>/SKILL.md`
```

## Чего скилл НЕ делает

- **Не редактирует SKILL.md.** Чтение, не правка.
- **Не запускает другие скиллы.** Это маршрутизатор справки, не диспетчер.
- **Не выдумывает поведение.** Если SKILL.md не содержит нужного раздела
  («Команда и флаги», «Триггеры») — сообщи это явно, не сочиняй замену.
- **Не показывает встроенные/системные скиллы Claude Code** (`find-skills`,
  `humanizer`, `loop`, `schedule` и т. д.). Только то, что лежит в
  `.claude/skills/` этого репозитория. Если пользователь спросил про
  системный скилл — кратко скажи, что справку даёт сам Claude Code.

---

## Краткие подсказки по парсингу SKILL.md

- `name:` и `description:` — в YAML frontmatter (между `---` сверху). description
  часто многострочный (`>` или `|`), читай как один блок.
- Триггеры обычно идут в самом конце description после слова «Триггеры:»;
  отдельный раздел «## Триггеры» — редкость.
- Раздел «Команды и флаги» — markdown-таблица сразу после `## Команда`,
  `## Команда и флаги` или `## Команды`. Возьми всю таблицу как есть.
